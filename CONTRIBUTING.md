# Contributing

Keep changes focused on making the finish gate more reproducible and useful to coding-agent UI work.

Good contributions include:

- verifier defects with a minimal reproduction;
- accessibility or state-coverage gaps;
- clearer evidence capture that does not add a service dependency;
- another small product workflow with explicit states and concrete actions.

Before opening a pull request:

1. explain the user problem, not only the preferred implementation;
2. keep the starter dependency-free unless maintainers explicitly approve otherwise;
3. run `npm run verify` against a completed implementation;
4. include a filled run log and authentic screenshots of the actual repository output;
5. disclose any relationship to a product mentioned in the change.

Do not submit promotional pull requests, generated marketing art, fabricated results, unrelated tool lists, referral links, or drive-by link additions.
